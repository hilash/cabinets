# Sources — Keto HQ

The trust layer for this cabinet. Every recommendation in every brief links back here. No file → no recommendation.

## When an agent cites something not in sources/

Write the source file first, then make the recommendation. If that's too much for a one-off, flag the suggestion as `[unsourced — needs source file]` so it can't be acted on until verified.

## Cross-references to Personal OS

When `personal-os/health/sources/` exists, Keto HQ agents may link to it for topics already covered there (e.g. magnesium glycinate, vitamin D, labs reference ranges). Don't duplicate.

## Evidence tier labels

- **strong** — multiple RCTs, meta-analyses, or established clinical guidelines.
- **emerging** — small trials, consistent observational data, strong mechanistic rationale.
- **anecdotal** — case reports, n=1 self-experiments, expert opinion.

Every source file declares its tier in frontmatter.
