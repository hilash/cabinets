# 🌱 Growth-Loop Generator

Every issue should have **one mechanism** that compounds reach. Not a generic "share this newsletter" footer — a specific, content-tied lever.

## Generator prompt

```
You are the Editor-in-Chief. Read brand/index.md and the target issue's draft.md.
Read growth-guide/index.md for the full lever menu.

Pick the ONE growth lever most appropriate to THIS issue's content. Don't stack
multiple — one strong CTA outperforms a stack.

Output:

  [LEVER NAME]
  One of: forward-friendly hook / reply-driven contribution / referral milestone /
  X or LinkedIn snippet / cross-promo invite / interview-the-reader call /
  archive deep-link.

  [WHERE IT GOES]
  Specify: "End of intro paragraph" or "Mid-issue, after section 2" or "P.S.
  after sign-off". Match the lever to the right placement.

  [THE COPY — paste-ready]
  The actual sentence(s) to drop into the draft. Voice must match brand/index.md.

  [WHY THIS LEVER FOR THIS ISSUE]
  One sentence reason. Not generic — tied to the issue's specific content.
```

## The lever menu (full descriptions in [[../growth-guide/index]])

| Lever | When to use |
|---|---|
| **Forward-friendly hook** | Issues with a sharp insight worth sharing — content does the work; ask is a soft P.S. |
| **Reply-driven contribution** | Open-ended issues where reader stories will become future content |
| **Referral milestone** | Once you've crossed 500+ subs and have rewards (paid issue, sticker, AMA) — Beehiiv / SparkLoop have this built-in |
| **X / LinkedIn snippet** | Issues that contain a punchy one-liner — drop it as a quote-image at the bottom for easy reposting |
| **Cross-promo invite** | When you have a relationship with another newsletter in the same niche — "5 newsletters I read every Sunday" type lists |
| **Interview-the-reader call** | When you need source material for upcoming issues — readers feel seen + you get content |
| **Archive deep-link** | Pull a high-performing past issue back into circulation — useful for new subscribers |

## What NOT to ask for

- "Please share with anyone you think would like it." — vague. Useless.
- "Don't forget to subscribe!" — they already are.
- "Like, share, and subscribe!" — you're not on YouTube.
- "Click here to upgrade to paid." — only the right time for the right issue. Most issues, no.

## The compounding move

The best newsletters compound because **every issue is forwarded**. To engineer that, every issue should have at least one moment a reader will want to send to a specific friend. Sometimes the issue itself does that work — the lever then is just a P.S. that says "if a friend sent you this..." with the subscribe link.

If the issue *doesn't* have a moment a reader would forward, the issue itself is weak. Don't try to growth-hack your way around weak content.
