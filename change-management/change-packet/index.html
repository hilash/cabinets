Change summary
Justification and scope
Summary
Upgrade the production PostgreSQL analytics cluster (3 nodes) from version 14.11 to 16.3 to address 4 CVEs (CVE-2024-10978, CVE-2024-10977, CVE-2024-10976, CVE-2024-7348) and gain parallel query improvements for the reporting workload.
Justification
PostgreSQL 14 reaches EOL November 2026. Two of the four CVEs are rated High (CVSS 7.5). Security team flagged this as a mandatory upgrade before the Q3 freeze. Performance testing shows 23% improvement in analytical query time on PG16 in staging.
Testing evidence
Full upgrade tested on analytics-staging cluster (identical schema, 30% of prod data volume) on 2026-05-19. Zero failures. pg_upgrade compatibility check: passed. 48h soak: no errors. Staging Datadog dashboards attached in GitHub PR #4471.
Comms plan
Engineering-wide notice 48h before window. Status page update at window start. Slack #analytics-ops live updates during migration. Stakeholder notification on completion: t.park → d.larsson (Analytics), s.okonkwo (CS).

Rollback plan
Decision criteria + specific recovery steps

Trigger criteria
Rollback if: (1) upgrade fails on any node and cannot be resolved within 30 min, OR (2) post-upgrade error rate on analytics-api exceeds 2% for 5 consecutive minutes, OR (3) replica lag exceeds 30s after primary is upgraded.
Recovery steps
pg_upgrade --old-datadir /var/lib/postgresql/14/main --new-datadir /var/lib/postgresql/16/main --revert → restart PG14 service → verify connections → update DNS/connection strings back to PG14 endpoint → notify stakeholders. ETA: 20 min.
Backups
