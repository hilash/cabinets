# ESC-041: API rate-limit errors returning 500 instead of 429

**Severity:** P1 — 23 affected accounts, 3 enterprise; documented API contract violated; automated workflows halted
**Status:** In Progress
**Created:** 2026-05-21
**Owner:** Platform Engineering (eng-platform@company.internal)
**Jira:** PLAT-2847

---

## Reproduction steps
1. Authenticate with a valid API token for any account on the Professional or Enterprise plan.
2. Make API requests at a rate exceeding the documented tier limit (300 req/min for Professional, 600 req/min for Enterprise).
3. **Expected:** HTTP 429 with `Retry-After` header per API documentation.
4. **Actual:** HTTP 500 with no `Retry-After` header, body: `{"error": "internal_error", "code": 500}`.
5. Reproduced on: REST v2 endpoint `/api/v2/records` (batch write). GET endpoints return 429 correctly. Bug is isolated to write endpoints under sustained load.

**Reproduction fidelity:** Confirmed — reproduced in staging by Platform Engineering on 2026-05-22.
