ESC-041 · P1 Bug Escalation
API rate-limit errors returning 500 instead of 429
P1 — Critical In Progress PLAT-2847 Platform Engineering Created May 21
Severity rationale

23 affected accounts across all tiers. Documented API contract violated — 500 returned instead of 429 on write endpoints, breaking customer retry logic. 2 Enterprise accounts in renewal within 60 days. Automated production workflows halted for at least 3 Enterprise customers.

Reproduction steps Confirmed
  1. Authenticate with any valid API token (all tiers affected).
  2. Target a v2 write endpoint: POST /api/v2/records or POST /api/v2/batch.
  3. Send requests at a rate exceeding the tier limit (300 req/min Professional, 600 req/min Enterprise).
  4. Expected: HTTP 429 with Retry-After header per API docs.
  5. Actual: HTTP 500 — {"error":"internal_error","code":500}, no Retry-After header.
  6. Note: GET endpoints return 429 correctly. Bug is isolated to v2 write endpoints under sustained load. V1 write endpoints also return 429 correctly.
