## Block A — Role Summary (Score: 4.3)

**What is the actual job?**
"PM, API Platform" at ElevenLabs means owning the external developer surface: the API design, SDKs, documentation, developer onboarding, usage-based billing, and the API console. ElevenLabs is an API-first company — the API platform PM is not a support function, it is a core growth function.

**Team & Reporting:**
- ~150-300 person company, growing fast
- Likely reports to VP Product or CPO
- Cross-functional with Engineering (infrastructure/API teams), DevRel, and Growth
- Small PM team (~5-10 PMs at current scale)

**Growth Potential: High.**
- API platform PM at a hyper-growth AI company creates rare career capital
- Path to "Head of Developer Experience" or "Director, Platform" within 12-18 months at their growth rate
- The ElevenLabs brand is a significant career asset in the AI PM market

**What the role actually entails:**
- Defining the API developer experience: authentication, SDKs, playgrounds, rate limits, model versioning
- Working with Growth on developer acquisition and activation (onboarding funnel ownership)
- Defining pricing and metering strategy for usage-based API
- Leading the interface between model team releases and developer-facing packaging

**Red Flags:**
- **AI hype risk**: ElevenLabs is growing at breakneck speed. Hype companies create hype roles — verify that this is a real PM function with engineering allocation, not a DevRel hybrid with a PM title.
- **Model-dependency**: The API platform PM at an AI company is dependent on model capability releases. If your roadmap is "whatever the model team ships next," scope is limited.
- **Title ambiguity**: "PM, API Platform" without a seniority qualifier. Clarify level — see Block C.
- **Company maturity**: ElevenLabs is moving very fast. Processes may be immature and expectations shifting.

"PM, API Platform" — no seniority signal. ElevenLabs, like many fast-growing AI startups, has not formalized leveling. The job probably lands at the Senior PM equivalent in scope, but this needs verification.

**Under-leveling risk: Low.**
API platform at a 150-300 person company is a significant scope. The role is unlikely to be junior — it likely requires someone who can operate without hand-holding.

**Over-leveling risk: Medium.**
If ElevenLabs is looking for an execution PM rather than a strategic PM, 8 years of experience might feel heavy. They may want someone more junior who will grind the developer feedback loop. Probe this in the recruiter call.

**Career Trajectory Fit:**
"PM Lead → Head of Product" trajectory is served well by this role if the scope is real. ElevenLabs would give you a 12-18 month window to become the most visible API platform PM in AI — then you're a Head of Product candidate anywhere.

## Block F — Interview Prep (Score: 4.2)

**Likely Format:**
1. Recruiter screen (30 min — culture + background)
2. Hiring manager interview (45-60 min — PM craft + background)
3. Product case: likely "design the ideal API developer onboarding experience" or "how would you improve our SDK?"
4. Cross-functional loop: engineering, growth, DevRel
5. Possible take-home (async product critique or written case)

**5 Probable Questions:**

1. "Walk me through a time you significantly improved developer onboarding." → **Story 2: SDK v3.0** — diary studies, co-design, time-to-first-API-call. This is your best story for this role.
2. "How do you think about API developer experience metrics?" → Lead with activation (time-to-value), retention (DAU/MAU of API keys), expansion (usage tier progression) — not just NPS.
3. "What would you change about the ElevenLabs API experience?" → Do your research. Use the API. File a mental bug list. Have 2-3 specific, concrete suggestions ready.
4. "How do you prioritize between developer feedback and the product roadmap?" → **Story 5: Feature That Got Killed** — shows you can say no with data.
5. "Tell me about a time you influenced a technical architecture decision as a PM." → **Story 4: Migration Nobody Wanted** (SOA migration with the strangler fig pattern).

**Company Intel to Weave In:**
- ElevenLabs has a "professional voice cloning" API and a "speech synthesis" API — know the distinction and have a view on which is the growth driver
- They launched a Conversational AI product in 2024 — understand how it relates to the core API
- The API pricing model is per-character — have a view on whether that's the right model vs. per-request or per-minute
- Their Discord has 200K+ members — the developer community is real and active

**Red Flag Questions to Ask:**
- "Does the API platform PM have engineering allocation, or do we compete for it?" (tests whether this is a real PM role)
- "How does the model team communicate upcoming capability changes to the API team?" (tests process maturity)
- "What's the biggest API developer pain point today?" (tests whether they're honest about their weaknesses)
- "What does the API revenue look like as a share of total revenue?" (tests whether API is a priority or a feature)
