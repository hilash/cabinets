# Manager 1:1

One persistent workspace per direct report — recurring agenda, running notes timeline, open action items, goals tracker, and a private feedback log. Two agents prep each 1:1 automatically and chase open action items weekly, so managers arrive informed instead of improvising and follow-through actually happens.

> **Keep Google Calendar and your HRIS. Replace the Notion page that gets updated once a quarter, the scattered Google Docs that never get opened before the meeting, and the action items that live only in someone's memory.**

## The team

- **[[.agents/people-ops]]** — People Ops Lead. Owns the 1:1 cadence — preps each session, synthesizes notes, and identifies patterns across sessions (recurring blockers, growth themes, stalled action items).
- **[[.agents/workflow-coordinator]]** — Workflow Coordinator. Executes the operational tasks around 1:1s: creates agenda docs, logs action items, sends follow-up summaries, flags overdue items.

## Recurring rhythm

| Cadence | Job | Owner | Output |
|---|---|---|---|
| **Weekly** (Mon 08:00) | [[.jobs/weekly-one-on-one-prep]] | People Ops Lead | Prep doc for each report with upcoming 1:1 this week |
| **Weekly** (Mon 08:00) | [[.jobs/weekly-action-item-followup]] | Workflow Coordinator | Action item status check; overdue items flagged to manager |

## How to run the demo

1. Open the [[one-on-one-room]] — the 1:1 room for a demo direct report: recurring agenda, recent notes timeline, open action items, goals tracker, and feedback log.
2. Browse [[reports/]] — one directory per direct report with their full 1:1 history.
3. Check [[reports/sam-patel/]] — the demo 1:1 workspace for Sam Patel (Software Engineer).

## Connectors

**Required:** Google Calendar (recurring 1:1 events), HRIS (roster, goals if tracked there), Google Docs / Drive (notes archive).
**Recommended:** Linear / JIRA (linked action items from project work), Lattice / Leapsome (goal tracking), Slack (async updates between sessions).

## HR data note

Demo notes use invented names and placeholder content. In production, 1:1 notes are manager-private — access is the manager and HR. No notes are shared with the report without explicit manager action.
