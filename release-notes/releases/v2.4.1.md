# Release v2.4.1 — May 21, 2026

Release owner: Mira Osei
Release date: 2026-05-21

---

## Customer-Facing Release Notes

### Features
- **Passkey authentication (WebAuthn).** You can now sign in using Face ID, Touch ID, or a hardware security key — no password required. Enroll a passkey from Security Settings → Authentication Methods. Works across all browsers with WebAuthn support (Chrome 108+, Safari 16+, Firefox 119+).

### Improvements
- **API rate limiting per workspace.** Rate limits now apply per workspace, not per API key, so hitting a limit in one workspace no longer affects others. Limits are visible in the API settings page. Workspace admins can request a limit increase from the settings.
- **Faster events dashboard.** The main events table now loads up to 62% faster for workspaces with large event histories. No action required.
- **Auto-generated API docs.** The API reference at docs.example.com now updates automatically with every release — no more stale examples.

### Fixes
- Fixed an issue where users were occasionally logged out unexpectedly after approximately one hour of inactivity. This affected single-page-app sessions that relied on background token refresh.

### Improvements
- **ENG-5391 — Per-tenant rate limiting.** PRs #1838–#1845 (8 PRs). Authors: Nadia Volkov, Jae-won Park. Extracted `AuthMiddleware` to `internal/middleware/auth.go`. Rate limits stored in Redis with 15-min TTL. Config in `config/rate_limits.yaml`. Support: new rate limit exceeded error code `429_WORKSPACE_LIMIT`.
- **ENG-5414 — events.tenant_id index.** PR #1853. Author: Lucas Ferreira. Migration `20260521_add_events_tenant_idx`. p99 query latency: 840ms → 320ms on production dataset. Zero-downtime migration (concurrent index build).
- **ENG-5409 — OpenAPI auto-gen.** PR #1854. Author: Lucas Ferreira. Added `go:generate` step in `Makefile`; CI now fails if spec is stale. Docs pushed to `docs.example.com` on every merge to main.

### Fixes
- **ENG-5388 (linked) / Bug report BUG-2291 — Session fixation / token refresh.** PR #1848. Author: Mira Osei. Root cause: `access_token` not rotated on SPA silent-refresh. Fix: invalidate old token before issuing new one. Affected: all SPA sessions using `grant_type=refresh_token`. Rolled out with session migration; no forced logout.
